## 初级功能测试岗位怎么看招聘要求

**哪些要求是真门槛**
- 熟悉软件测试流程和方法论（等价类、边界值、场景法）— 必须掌握
- 会用 JIRA / 禅道等缺陷管理工具 — 必须掌握
- 能编写测试用例和测试报告 — 必须掌握
- 基础 SQL 查询 — 必须掌握
- 基础接口测试（Postman / F12）— 必须掌握

**哪些要求可以边学边补**
- 自动化测试（Python/Selenium）— 进阶方向，新人岗通常不强要求，入职后学
- 性能测试（JMeter）— 同上
- 精通某行业领域（电商/金融/医疗）— 工作中积累即可
- 懂 Git / CI/CD — 加分项，但非硬门槛

**招聘信息里的水分识别**
- "1年经验"很多公司招应届生时也写这个，不要被吓跑
- "精通XX"通常意味着"会用就行"，真精通的另有薪资

## 简历怎么写

**零基础转型简历的常见误区**
转行者简历最容易犯的错误是：罗列词汇、缺乏产出、像一个说明书。

❌ 坏的项目描述：
> "负责电商系统的登录、购物车、支付模块的测试。使用等价类、边界值设计用例，用 Postman 测试接口，用 JIRA 报 Bug。"

这种描述等于没写，完全没有区分度。

**用 STAR 法则写项目经验**
STAR = Situation（情境）、Task（任务）、Action（行动）、Result（结果）

✅ 好的项目描述（以同一件事为例）：
> **核心行动与产出：**
> - 主导电商项目 C 端（登录、支付核心订单链路）的功能与接口测试。针对历史遗留的超卖与库存并发逻辑设计深度测试用例。
> - 在测试执行周期内发现并追踪超过 45 个有效缺陷，其中包含 3 个支付阻断性的 P0 级重大 Bug，并在 JIRA 中完成全生命周期闭环管理。
> - 使用 Postman 完成上下游接口的联调验证，编写 MySQL 语句造库查询脏数据，通过 Chrome DevTools 抓包精准定位 20+ 起前后端对接问题，提高了修复效率 15%。

关键要素：**有数字**（45个缺陷、3个P0）+ **有工具**（Postman、MySQL、DevTools）+ **体现解决了什么问题**。

## 面试怎么讲项目

**如何讲测试流程**
面试官问"你在这个项目里是怎么测的？"

> "我们采用敏捷迭代开发模式，两周一个 Sprint。我主要负责 XX 模块。在需求评审阶段我会仔细阅读 PRD，提出疑问点；开发并行期间同步编写测试用例；提测后对照用例执行，发现的 Bug 在 JIRA 里提单跟踪。这个迭代我一共执行了 X 条用例，发现 Y 个有效 Bug，全部关闭后输出了测试报告，建议上线。"

**如何讲 Bug**
面试官问"说一个你印象深刻的 Bug"

> "我在测试支付流程时发现，当用户在支付确认页断网，然后重新联网后，系统会弹出支付成功的提示，但实际上订单处于未支付状态，且数据库中没有支付记录。我通过 F12 抓包发现是前端缓存了支付结果而没有向后端再次确认。这是一个 P1 级的严重 Bug，我在 JIRA 中详细记录了复现步骤和抓包截图，开发在两天内完成了修复。"

**如何讲接口测试入门能力**
> "我会用 F12 的 Network 面板在发现 Bug 时先排查是前端还是后端的问题。对于需要绕过前端界面直接测后端逻辑的场景，我会用 Postman 构造请求，验证接口的状态码、返回字段格式，以及异常参数时是否有合理的错误提示。"

## 常见面试题整理

**测试基础题**

Q：软件测试的目的是什么？
> 发现软件中的缺陷，验证软件符合需求，为上线提供质量评估依据。

Q：黑盒测试和白盒测试的区别？
> 黑盒测试不关注代码内部实现，只关注输入和输出是否符合需求；白盒测试需要看代码结构，针对代码逻辑进行测试。

**经典必考题：开发不认 Bug 怎么办？**

❌ 错误回答：跟他吵，或者直接上线。

✅ 满分思路：
1. 先自我排查，确保复现步骤没有问题
2. 拿 PRD 或接口文档中的约定作为依据，客观说明为什么这是 Bug
3. 如果 PRD 没写清楚，立刻拉上产品经理三方碰头确认，以产品最终认定为准，并更新到文档中

**经典必考题：明天上线，今天发现一个难修的 Bug，同意上线吗？**

✅ 满分思路：
- 不能一概而论，要评估 Bug 的**严重程度和影响范围**
- P0 阻断级 Bug（主链路崩溃、资金异常）：坚决拦截，立刻报备，紧急修复后再发
- P3 轻微 Bug（文案错误、非核心 UI 问题）：可以提"延期修复"单，由产品经理和项目经理评估认可后带病上线，排期到下个版本修复
- 无论哪种情况，**绝不私自隐瞒**，必须有书面留痕

**用例设计方法题**

Q：等价类划分法和边界值分析法的区别？
> 等价类是把输入数据分成有效和无效的类，每类只测一个代表值；边界值是专门测边界上的那几个特殊值（如最小值-1、最小值、最大值、最大值+1）。两个方法通常配合使用。

## 甲方 vs 外包如何选择

**外包公司**（如中软、软通、文思海辉等驻场）
- 门槛相对较低，面试往往较快
- 优点：可以快速获得大厂业务的项目经验，丰富简历
- 缺点：归属感差，核心架构或自动化往往接触不到，主要做纯功能黑盒测试

**甲方公司（自研团队）**
- 自己研发自己的产品，测试有一定的话语权和成长体系
- 面试门槛相对高一些，竞争者更多

**结论**：作为零基础转型者，**不要太排斥外包**。生存第一——如果进不去好的自研团队，第一份工作在外包干 1-2 年打好业务基本功，积累项目经验，把它作为跳板再进更好的公司，是完全合理的路径。
