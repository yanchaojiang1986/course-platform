# 模块 10：入职与成长

## 学习目标

- 拿到 Offer 后，你将知道第一个月怎么过
- 了解职场高压线和避坑法则
- 建立 1-3 年的职业进阶方向认知

---

## 新人入职 30 天落地生存指南

拿到 Offer 之后，千万不要一进来就想着马上写高级用例、发现惊天 Bug。新人的第一目标是**"活下来，融入环境"**。

**第 1-3 天：配环境与读文档**
- 这三天可能连开发库都连不上。任务是找导师或组长，把账号体系全部搞定：公司邮箱、内网权限、JIRA 账号、开发测试环境访问权
- 尽一切可能收集历史的产品需求说明书、测试用例表；去缺陷管理库里翻看以前大家常提什么 Bug
- 把内部常用的行业黑话和系统模块名称记录下来

**第 4-14 天：模仿与执行现有测试用例**
- 通常导师不会直接交给你一个全新的大型需求，而是丢给你一个已有的测试用例集合，比如新版本上线前的"例行回归测试"
- 照着别人写好的步骤一步步点软件，熟悉真实业务的操作流——这是最高效的业务上手方式
- 执行过程中遇到不确定的现象，先记录下来，再去问旁边的老测试

**第 15-30 天：承担小模块**
- 开始接手一两个小页面的独立需求
- 不确定是不是 Bug 时，先问问旁边的老测试，不要轻易提单被开发怼
- 每天下班前整理笔记，复盘遇到的坑
- 30 天后，你能成为该小型业务模块的熟手

## 核心职场高压线与避坑法则

**高压线一：绝对不要乱碰生产环境数据库！**

测试人员会有大量时间操作数据库。如果你拿到了生产环境（真实用户使用的机器）的数据库权限，连错了数据库执行了 `UPDATE` 不加 `WHERE` 条件，或者删了表，可能造成极大的线上事故。

所有测试行为**必须**在特定的测试数据库中进行。每次执行 SQL 前，先确认当前连接的是哪个数据库。

**高压线二：不接口头需求和私活**

开发走廊端着咖啡对你说："我稍微改了几行小代码，影响不大，你随便帮我点一下我们发上线了"。

**绝对不要答应！** 任何改动必须走规范的提测流程，进行正式的测试范围评估。很多线上 P0 事故都是这种"稍微改几行"引发的。测试人员是容易背锅的群体，一切需留痕处理。

## 从功能测试到更高阶的进阶路径

只有功能测试的基础，1-2 年后会遇到职业天花板。当你站稳脚跟后，需要朝以下方向突破：
<!-- CHECK:q10_3 -->

### 路径一：全栈自动化工程师（技术方向，最主流）

- **成长路径**：从使用界面工具，转向利用代码编写自动化脚本，大幅提升测试效率
- **学习内容**：
  - Python / Java 语言核心编程
  - **Selenium / Playwright**：实现自动打开浏览器点击页面的 Web 自动化
  - **Appium**：实现手机界面的自动滑动、点击
  - **Pytest / TestNG + Jenkins**：构建每天晚上自动运行、次日发送全系体检报告的 CI/CD 自动化流水线

### 路径二：测试开发工程师 / 研发效能专家（高薪方向）

- **成长路径**：不再局限于测试软件，而是开发一套工具或平台，专门提升测试团队的 Bug 发现效率
- **特点**：需要等同于中级甚至高级后端的代码能力，但薪资上限高

### 路径三：业务专家与测试管理（软技能方向）

- **成长路径**：对电商/支付/金融/医疗等特定行业的业务极其熟悉，凭借大局观与协调能力走向测试架构、测试经理乃至测试总监
- **特点**：不需要极强的编码能力，但需要深度的行业理解和优秀的团队管理能力

## 10.4 入职 1 年内的硬核成长路线图

很多人入职后陷入"日常修 Bug—下班—第二天继续修 Bug"的循环，1 年后水平和入职时差不多。**主动设计成长路线才能不被淘汰。**

**入职 1 年成长 4 阶段**

<!-- DEMO:10-03 -->

### 第 0–1 个月：生存期

**目标**：能独立跑完一个版本的测试，不出大乱子。

**任务清单**：
- [ ] 跟着导师测完 2 个迭代
- [ ] 熟悉公司的缺陷管理工具（JIRA / 禅道）
- [ ] 阅读至少 100 个历史 Bug，建立对系统的认知
- [ ] 与开发、产品建立基础信任

**重要事项**：**别问太多基础问题**——能 Google / 问 AI 的别问人；同一个问题别问第二次。

### 第 2–3 个月：建立资产期

**目标**：建立自己的"测试资产"——可复用的模板、清单、脚本。

**任务清单**：
- [ ] 整理一份《公司测试规范精简版》（用例模板、Bug 单模板、报告模板）
- [ ] 建立一个《模块知识库》（每个模块的业务逻辑、关键接口、常见 Bug）
- [ ] 学习公司用的所有工具（如 Charles、Postman、SQL 客户端）

**重要事项**：**让别人感受到你的输出价值**——把你整理的知识库分享给同事，会显得你有 Owner 意识。

### 第 4–6 个月：能力深化期

**目标**：成为 1-2 个模块的"专家"，开发遇到问题首先找你确认。

**任务清单**：
- [ ] 主动接手 1-2 个核心模块的全部测试
- [ ] 学习 Python 基础（每天 1 小时，3 个月入门）
- [ ] 开始尝试用 Postman + Newman 做接口自动化

**重要事项**：**主动汇报，建立可见性**——周会、月会主动分享你的发现和改进，别人才会知道你的价值。

### 第 7–12 个月：突破期

**目标**：开始向自动化测试或测试开发转型。

**任务清单**：
- [ ] 用 Selenium + Pytest 写 1 套 Web 自动化用例
- [ ] 用 Appium 写 1 套移动端自动化用例（移动端项目可选）
- [ ] 部署 Jenkins，做每日自动跑回归
- [ ] 输出 1 份《年度测试效率提升报告》

**重要事项**：**做出可量化的成果**——"自动化让回归时间从 8 小时降到 2 小时"比"我学了 Selenium"有说服力 10 倍。

## 10.5 从功能测试到自动化的转型路径

很多人卡在"功能测试做了 2 年，想转自动化但没机会"。这里给出完整路径：

**第一步：在本职工作中"抠"出实践机会**

- 用 Postman Collection Runner 跑回归（不需要写代码）
- 用 Newman 把 Postman 集合接入 Jenkins
- 用 Python + Pytest 替换部分手动验证

**第二步：业余项目积累代码**

- GitHub 上找开源项目，给它写自动化用例
- 自己开发一个小工具（如：批量造测试账号的脚本）

**第三步：内部转型**

- 主动申请承担团队的自动化建设任务
- 把节省的时间用于自我培训，形成正向循环
- 用数据证明 ROI，向上争取转岗

**第四步：跳槽到自动化岗位**

- 简历里增加自动化项目占比
- 面试时重点讲"如何用代码提升测试效率"
- 期望薪资可上调 30-50%

## 10.6 中级岗位的核心要求（提前看下一阶段目标）

了解中级岗位要什么，反向指导你现在该学什么。

**中级测试工程师岗位 JD 拆解**

| 能力维度 | 初级要求 | **中级要求** |
|---|---|---|
| 测试方法 | 会写用例 | **能独立做模块测试方案设计** |
| 编程能力 | 会读 SQL | **会写 Python / Java，能搭自动化框架** |
| 工具使用 | 用 Postman / JIRA | **会写 JMeter 性能脚本、用 Jenkins 搭流水线** |
| 系统思维 | 测好单个功能 | **能看懂系统架构、识别系统性风险** |
| 协作能力 | 会提 Bug | **能 review 别人的用例、培训新人** |
| 项目管理 | 跟着计划走 | **能制定测试计划、把控版本质量** |

**中级岗位面试重点**

1. 自动化框架设计（不只是会用，是会设计）
2. 性能测试实战（不只是看报告，是能定位瓶颈）
3. 项目管理（如何把控时间、人员、质量）
4. 团队协作（如何带新人、如何说服开发）

**给现在的你的建议**：从入职第一天起，就**有意识地朝中级方向准备**。每完成一个测试任务，问自己一句"如果是中级测试，会怎么做？"
