## 项目 A：Web 电商平台测试

**背景说明**
系统包含面向普通用户的 Web 端商城，以及面向商家的 Web 端管理后台。这是国内最典型的业务流，也是面试中最常被问到的项目类型。

**核心测试需求提取**

**注册/登录模块**
- 不仅要测空账号无法登录，还要测"短时间频繁获取短信验证码会不会被风控锁定"
- 第三方登录（微信/手机号一键登录）失败后的兜底提示是什么
- 密码重置流程中，旧密码是否立即失效

**商品浏览与购物车的联动**
- 商品库存是有限的。重点测试：当你加入购物车后，是在这一刻扣减库存，还是在最终支付成功才扣库存？
- "占库存分析"：恶意用户反复加购物车不买，会不会导致别人买不到？（是否在加购时占库存、下单时占库存或支付前锁库存，取决于具体业务策略）
- 购物车中的商品下架后，结算时如何提示？

**支付链路（核心场景法应用）**
- 在测试环境没有真实货币，通常使用支付宝沙箱环境或内部挡板接口模拟
- 弱网测试：在输入密码的瞬间断开网络，系统是超时报错、保留订单，还是陷入死循环？
- 重复支付：同一订单能否被支付两次？
- 支付成功后立刻申请退款，退款金额是否正确？

> **课后作业**：为"商品库存超买"场景编写至少 5 条边界值与异常处理关联的测试用例。

## 项目 B：移动端社交 App 测试

移动端 App 测试不仅有功能逻辑的要求，还有依附于手机特性的测试，是当前最重要的测试方向之一。

**消息发送基础功能**
- 正常发送文字、图片、视频、文件
- 发送后消息是否正确显示（时间戳、已读状态）
- 消息撤回是否在双方都生效，撤回时间限制

**交叉事件中断测试**
- 正在上传多张图片发朋友圈，突然来了一个电话或闹钟响了
- 退回桌面，手机内存不足把 App 清理了，重新打开后草稿箱里的数据还在吗？

**弱网与网络切换测试**
- 把网络限制成 2G/3G 状态，观察图片上传的表现
- 在 Wi-Fi、4G/5G、断网状态之间切换时，系统有没有友好的提示，还是直接崩溃？

**安装、卸载与升级测试**
- 覆盖安装测试：当前装了老版本 V1.0，直接安装新版本 V2.0，以前的聊天记录和设置项有没有丢失或错乱？

**系统权限管理测试**
- 第一次要发图片，系统弹窗要求"相册"权限，用户点了"拒绝永久询问"
- 后续点击发布时，系统有没有提供友好的"去设置里打开权限"的引导，而不是默默失败？

## 项目 C：综合交付项目（从需求到报告的完整模拟）

这是最重要的一个项目，模拟真实工作环境中的完整测试流程。

**步骤一：需求理解**
- 拿到一份模拟 PRD（可以使用飞书或开源系统的文档）
- 参加模拟需求评审，提出至少 3 个隐性需求或逻辑漏洞

**步骤二：编写测试点与测试用例**
- 用业务场景树拆解需求
- 针对每个测试点，运用四大方法编写标准用例（目标：至少 30 条）

**步骤三：执行测试并提 Bug**
- 找一个真实的开源系统进行测试（推荐：GitHub 上带 docker-compose 的电商项目）
- 在禅道/JIRA（或表格模拟）中提交至少 5 条标准 Bug 单

**步骤四：写测试报告**
- 按模块 05 的格式写一份简版测试报告
- 给出明确的上线建议

**进阶建议**
- 在电脑上用 Docker 部署一套开源测试管理软件（如 Bugfree）
- 在 GitHub/Gitee 找一个带有安装文档的完整电商后端项目，把它跑起来
- 连上 MySQL 数据库查看后端存了什么，用 F12 抓取接口查看 JSON 字段
- 完成这一套，你已经远远超越那些"背背理论就去面试"的零基础求职者

## 复杂系统专项实战模板（银行网银 / 商业汇票 / 大型中台系统）

如果你未来进入金融、票据、支付、供应链等领域，测试对象通常是“强规则 + 多状态 + 多系统联动”的复杂系统。

**实战模板：先画 3 张图**
1. **业务域模块图**：账户域、交易域、审批域、对账域
2. **状态流转图**：例如汇票“出票→背书→贴现→到期→兑付”
3. **系统交互图**：客户端、业务中台、风控中台、账务系统、通知系统

**每个复杂系统至少覆盖 5 类关键场景**
1. 主流程成功（端到端）
2. 权限/角色冲突（A可见，B不可见）
3. 状态跳转非法（越级操作、逆向操作）
4. 外部系统失败（超时、拒绝、重复回调）
5. 对账差异处理（T+1 不一致定位）

**面试价值最高的表达方式**
- 不要只说“测了很多页面”
- 要说“我负责了哪个业务域、哪些状态机、哪些跨系统风险，以及如何控制”

## 08.5 三大场景实战沙盒（SVIP 专享 AI 陪练）

> 以下三个实战场景配套 Mock API 沙盒 + 苏格拉底 AI 陪练。SVIP 用户可以直接在教程内完成完整流程并获得 AI 评分。

### 🎯 实战场景 A：TechStart 注册功能测试

**业务背景**：TechStart 是一家创业公司，正在开发新的会员注册系统。你作为测试工程师，刚收到 PM 发来的 PRD：

> "用户通过手机号注册账号。手机号必须是中国大陆 11 位号码，密码 8-16 位含大小写字母和数字。注册时需短信验证码，5 分钟有效。注册成功后自动登录，跳转个人中心。"

**你的任务**：
1. 提炼至少 30 个测试点（按维度分类）
2. 设计标准测试用例（至少 20 条）
3. 找出 PRD 中至少 5 个隐性需求（如"同一手机号短时间内能注册几次"）
4. 在教程内置 Mock API 上实际跑 5 条核心用例

**SVIP AI 陪练**：
- AI 不会直接给你答案
- 你写完测试点后，AI 反问"为什么没考虑短信被劫持的场景"
- AI 给你的用例打分（覆盖度 / 边界完整性 / 异常处理）

**预期产出**：完整测试用例文档（可作为简历项目）

---

### 🎯 实战场景 B：电商购物车结算

**业务背景**：你接手了一个电商项目的购物车 + 结算模块。涉及多个角色和状态。

> "用户可以添加商品到购物车。结算时选择优惠券、地址、支付方式。订单状态有：待支付 / 已支付 / 已发货 / 已完成 / 已退款。库存在支付成功时扣减。"

**你的任务**：
1. 画出订单状态机图，标注所有合法跳转和非法跳转
2. 设计涉及优惠券、库存、支付的联合测试用例
3. 实现一个完整的 Mock API 接口测试
4. 输出完整测试报告（包含遗留 Bug 评估）

**关键测试维度**：
- 多商品 + 多规格 + 多优惠券组合
- 并发下单（库存幂等性）
- 支付中断后订单状态
- 退款后库存恢复

**SVIP AI 陪练**：
- AI 扮演产品经理，回答你提出的需求疑问
- AI 评审你的测试用例，指出遗漏的 5 个场景

---

### 🎯 实战场景 C：后台管理系统权限测试

**业务背景**：你负责一个 CMS 后台的权限模块测试，难度最高。

> "系统有 5 种角色：超管、内容管理员、运营、客服、只读账号。每种角色对资源（文章 / 用户 / 订单 / 系统设置）有不同的 CRUD 权限。"

**你的任务**：
1. 制作完整的权限矩阵表（5 角色 × 4 资源 × 4 操作 = 80 项）
2. 设计水平越权测试（A 客服能看 B 客服的工单吗？）
3. 设计垂直越权测试（普通管理员能调超管接口吗？）
4. 使用 Postman 修改 token 实际验证

**关键测试维度**：
- 通过 URL 直接访问无权限页面
- 通过修改请求参数访问其他用户资源
- 角色变更后的权限实时生效
- 离职用户账号禁用后的会话失效

**SVIP AI 陪练**：
- AI 扮演开发，回答你关于权限实现的技术问题
- AI 检查你的越权测试是否覆盖了 OWASP Top 10 中的相关项
